## Codebase Patterns
- PHPStan level 8: use `/** @var resource */` or helper methods to narrow types from nullable properties after guard methods
- Sender interface requires: `post(string $endpoint, string $apiToken, array $payload, FlareEntityType $type, bool $test, Closure $callback): void`
- DaemonSender at `src/Senders/DaemonSender.php` - implements TCP payload protocol with format `{total_length}:v1:{type}:{jsonPayload}`
- DaemonTimeoutException at `src/Senders/Exceptions/DaemonTimeoutException.php` - thrown when test response times out
- FlareConfig uses fluent builder pattern with `static` return types
- ConnectionError exception wraps all sender connection failures
- Code style: always use curly braces, happy path last, avoid else
- Tester class at `src/Support/Tester.php` - test() orchestrates sending test payloads for all FlareEntityType cases
- FlareProvider at `src/FlareProvider.php` - wires singletons via Container, must update when constructor params change
- Telemetry at `src/Support/Telemetry.php` - provides SDK name/version via Composer InstalledVersions
- Daemon package at `daemon/` is a separate Composer package (`spatie/flare-daemon`) with namespace `Spatie\FlareDaemon\`
- Daemon uses ReactPHP: event-loop, socket, http, promise, stream
- Root `.gitignore` patterns (`vendor`, `composer.lock`) match at any level, covering `daemon/vendor/` too
- Daemon PHPStan runs from `daemon/` dir: `vendor/bin/phpstan analyse --configuration=phpstan.neon.dist`
