## Codebase Patterns
- PHPStan level 8: use `/** @var resource */` or helper methods to narrow types from nullable properties after guard methods
- Test fakes at `daemon/tests/`: LoopFake (deterministic timers), BrowserFake (HTTP recording), Connection (TCP mock), SyncedClock (time-synced with LoopFake), OutputWriterFake (captures output)
- LoopFake implements both `LoopContract` and `LoopInterface` â€” use `@param` docblocks on untyped LoopInterface methods for PHPStan
- LoopFake `advance(seconds)` fires timers deterministically â€” no real I/O or sleep
- BrowserFake `queueResponse(urlPattern, response)` matches by str_contains â€” use URL fragments like `/v1/errors` or `/v1/usage`
- All daemon production classes accept `LoopContract` (not concrete `Loop`) â€” enables LoopFake injection without runtime type errors
- OutputWriterFake extends OutputWriter with empty constructor â€” use for tests instead of real OutputWriter
- Server feature tests use reflection to call `handleConnection()` on the Server with a fake Connection â€” bypass real TCP listener
- PHPStan `// @phpstan-ignore argument.type` must be on the exact line of the argument, not the function call opening line
- PHPStan level max: `write($data)` from ConnectionInterface receives `mixed` â€” use `is_string()` guard, not `(string)` cast
- Sender interface requires: `post(string $endpoint, string $apiToken, array $payload, FlareEntityType $type, bool $test, Closure $callback): void`
- DaemonSender at `src/Senders/DaemonSender.php` - implements TCP payload protocol with format `{total_length}:v1:{type}:{jsonPayload}`
- DaemonTimeoutException at `src/Senders/Exceptions/DaemonTimeoutException.php` - thrown when test response times out
- FlareConfig uses fluent builder pattern with `static` return types
- ConnectionError exception wraps all sender connection failures
- Code style: always use curly braces, happy path last, avoid else
- Tester class at `src/Support/Tester.php` - test() orchestrates sending test payloads for all FlareEntityType cases
- FlareProvider at `src/FlareProvider.php` - wires singletons via Container, must update when constructor params change
- Telemetry at `src/Support/Telemetry.php` - provides SDK name/version via Composer InstalledVersions
- Daemon package at `daemon/` is a separate Composer package (`spatie/flare-daemon`) with namespace `Spatie\FlareDaemon\`
- Daemon uses ReactPHP: event-loop, socket, http, promise, stream
- Root `.gitignore` patterns (`vendor`, `composer.lock`) match at any level, covering `daemon/vendor/` too
- Daemon PHPStan runs from `daemon/` dir: `vendor/bin/phpstan analyse --configuration=phpstan.neon.dist`
- Daemon tests run from `daemon/` dir: `vendor/bin/phpunit`
- After adding new classes in daemon/, run `composer dump-autoload` (classmap-authoritative mode)
- Payload class at `daemon/src/Payload.php` - TCP protocol parser with chunked data support and overflow handling
- StreamBuffer at `daemon/src/StreamBuffer.php` - per-type payload accumulator with ~6MB flush threshold
- NullBuffer at `daemon/src/NullBuffer.php` - no-op buffer that drops payloads (used for over-quota types)
- Server at `daemon/src/Server.php` - TCP server accepting persistent connections, routing payloads via closures
- Server uses closure-based routing: `$onPayload(type, data)`, `$onTestPayload(baseType, data, connection)`, `$onStatus()` â€” decoupled from Ingest
- PHPStan level max: assign `new TcpServer()` to a local variable before calling `on()` â€” assigning to nullable `$this->server` first causes method-on-null errors
- Ingest at `daemon/src/Ingest.php` - manages per-type StreamBuffers, HTTP forwarding with gzip, response handling, test payload routing
- Ingest uses closure callbacks for cross-concern communication: `onUsageIncrement`, `onInvalidApiKey`, `onQuotaExceeded`
- Ingest test payloads: always write to realBuffers (bypassing NullBuffer), flush individually (not batched), track pending test connections by payloadId
- Ingest `forceDigest()` returns a Promise that resolves when all in-flight requests complete (uses periodic timer polling)
- Ingest `forceDigest()` also calls `flushRealBuffersForPendingTests()` to ensure test payloads in real buffers get sent even when types are paused
- Ingest stores test payload `data` in `pendingTestConnections` for re-identification during force-digest flushes
- UsageRepository at `daemon/src/UsageRepository.php` - fetches quota from Flare, tracks local counters, pauses/resumes types via Ingest
- UsageRepository hooks into Ingest via `onUsageIncrement` and `onQuotaExceeded` callbacks set in `start()`
- Usage value object at `daemon/src/Usage.php` - `fromArray()` factory parses API response, helper methods `isOverLimit()`, `allOverLimit()`, `used()`, `limit()`
- PHPStan level max: `json_decode` returns `mixed` â€” use `/** @var array<string, mixed> */` annotation after `is_array()` guard for `fromArray()` calls
- PHPStan level max: cannot cast `mixed` to int/string â€” use explicit type checks (`is_int`, `is_numeric`, `is_string`) in value object factories
- CheckForUpdates at `daemon/src/CheckForUpdates.php` - monitors composer.lock hash every 60s, initiates 5-min graceful shutdown on change
- PHPStan level max: `Ingest::status()` returns `array<string, mixed>` â€” use `/** @var array<string, array<string, mixed>> */` annotations to narrow nested access in tests
