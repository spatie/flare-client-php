# Ralph Progress Log
Started: Wed Feb 25 16:00:58 CET 2026

## Codebase Patterns
- `FlareConfig` uses constructor property promotion with fluent setter methods returning `static`
- `FlareProvider` resolves `FlareMode` via match expression in constructor — add new mode cases there
- `FlareMode::Disabled` is explicitly checked in `FlareProvider` for disabling tracer, logger, and reporter — new modes that should be "enabled" just need to not match `FlareMode::Disabled`
- `Sender` interface requires implementing `post()` with specific signature including `FlareEntityType`, `bool $test`, and `Closure $callback`
- Senders accept `array $config` in constructor and extract their specific config keys from it
- Tests use Pest, run with `composer test`; PHPStan at level 8 via `composer analyse`
- `Api` is constructed in `FlareProvider::register()` — new constructor params must be wired there from `$this->config`
- No Mockery available — use anonymous classes implementing interfaces for test doubles
- `psr/log` is available transitively through `monolog/monolog` — `LoggerInterface` can be used
